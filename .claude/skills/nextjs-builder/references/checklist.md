# 5軸レビューチェックリスト

実装の**前**（設計時）と**後**（提出前）に通す。1つでも欠けたら直すか、理由を明示する。

## 実装前（設計チェック）
- [ ] 既存規約を調べたか？（PM・Next バージョン・DB・認証・スタイリング・ESLint 設定）
- [ ] server / client の境界をどこに置くか決めたか？`'use client'` は葉に押し下げたか？
- [ ] データ取得の場所とキャッシュ/再検証戦略を決めたか？
- [ ] 信頼境界（Server Action / Route Handler）はどこか、そこで何を検証・認可するか決めたか？
- [ ] 新規ライブラリを勝手に導入していないか？（既存で足りるか確認したか）

## 実装後（5軸レビュー）

### ① 可読性
- [ ] 命名が意図を語るか？マジック値は定数化したか？
- [ ] 早期リターンでネストが浅いか？関数/コンポーネントは短く単一目的か？
- [ ] コメントは「なぜ」に限られているか？死んだコードは無いか？
- [ ] 既存の Prettier/ESLint スタイルに合っているか？

### ② ドメイン適合（RSC イディオム）
- [ ] Server Component をデフォルトにし、client 化は必要な葉だけか？
- [ ] ルーティング予約ファイル（`page`/`layout`/`loading`/`error`/`route`）を正しく使ったか？
- [ ] メタデータは `metadata`/`generateMetadata` で扱ったか？
- [ ] ページ描画のために自分の Route Handler を fetch していないか？

### ③ パフォーマンス
- [ ] 独立データ取得を `Promise.all` で並列化したか？重複は `React.cache` か？
- [ ] 遅い領域を `<Suspense>` でストリーミングしたか？
- [ ] `next/image` / `next/font` を使ったか？重い client 依存を `next/dynamic` 化したか？
- [ ] クライアントに不要な大きな props / JS を送っていないか？

### ④ セキュリティ
- [ ] 全境界入力を Zod 検証したか？
- [ ] Server Action / Route Handler の先頭で認証・認可（所有チェック含む）したか？
- [ ] 秘密がクライアントに漏れていないか？秘密モジュールに `server-only` を付けたか？
- [ ] `dangerouslySetInnerHTML` / 文字列連結SQL / オープンリダイレクト / SSRF を避けたか？

### ⑤ SOLID
- [ ] 各コンポーネントは単一責任か？（取得・表示・状態を混ぜていないか）
- [ ] props は最小限か（god props でないか）？
- [ ] UI がデータ層の抽象越しに依存しているか（具体実装を直接埋め込んでいないか）？
- [ ] `any` を排し、不正状態を型で表現不能にしたか？

## 最終確認（可能なら実行）
- [ ] 型チェック（`tsc --noEmit` 相当）が通るか？
- [ ] リンタ（ESLint）が通るか？
- [ ] ビルド（`next build`）が通るか？
- [ ] 生成コードは省略なしで**そのまま動く**か？
